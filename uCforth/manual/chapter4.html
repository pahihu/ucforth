There are many,many,many HTML tags. If you want your document to have a long life and a wide audience,keep it simple. The follow gives you a nice selection.

In most case you have a start tag for example: <p>, and an end tag for example: </p>. Some browser allow you to be slack and leave off the end tag, don't. The shortcuts will not work across all browsers and the standard teams are trying to stamp it out. The golden rule. Always close your tags.

General

Keep you files small. Some browser don't start displaying the content until all the data has arrived. The aim is to interest, not to bore.

HTML Editor

Windows installs NOTEPAD ( a simple text editor) as the default HTML editor. It is a good choice. Start and end your document with the header and footer discussed below. Save it with a .html extension and open it in your browser. When a section is completed save the file again and reload into your browser.

Document start

A html document has a header and a body. Search engines are controlled by robots, if you want to be listed first you look after the robots. Robots roam the internet retrieving and indexing content. Lazy robots only look at the header. They are interested in three things. keywords author and title. Get them right.

The header is placed at the start of the document. The html code for a simple header follows. 


<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 3.2//EN">
<HTML>
<HEAD>
<TITLE>Title of your document</TITLE>
        <META NAME="AUTHOR" CONTENT="Your name">
        <META NAME="keywords" CONTENT="Search words">
</HEAD>

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 3.2//EN"> tells the browser what sort of code you are sending it. <HTML> Is an opening tag that tells the browser we are going to talk html. This tag is closed at the end of the document. <HEAD>is the opening tag for the header section. The header section contains the three pieces of information used by the robots and is ended with the closing tag </HEAD>. We then open the body section with the tag <BODY>.

Document end

We end the file with tags to close the body section and the html code. 


</BODY>
</HTML>

Content

Between <BODY> and </BODY> you put your content. Content is text, images, a few tags to tell the browser how to lay it all out, some code for the COLDFORTH compiler and links. HTML tags are just like english words; 95% of them are of no interest, available for those that have nothing better to do.

Formatting

Divide your text into paragraphs.

<p>
This is paragraph text.
</p>

The browser will take the text between <p> and </p>, remove all the carriage returns and treat the data as a block of text to be displayed. If you really want a carriage return you have to add the tag <br>, a "bloody return". There is no matching closing tag, that is the way computer science is, there is always an exception.

Images

A browser can display many different image formats; only three are really of interest.

.png is the preferred form. Unfortunately .jpeg and .gif have better support. .jpeg ( for pictures) and .gif ( for graphics). .gif is tied up with patent issues. If you use them you take a risk.

Take a look at the size of your picture files; if they are over 64k don't use them, no-one is going to wait for them to download.

A browser can lay out the text before the images arrives, if you provide the image size.

Provide alternative text. This is needed if the image can't be found, if the browser can't display the image type you have used and it is useful as it gives a hint as to what is coming while only the text is displayed.

Always provide the image size; The image size is given by all good image editors. 


<img src=" ./images/image_name.png" width="277" height="300" alt="text displayed instead" >
